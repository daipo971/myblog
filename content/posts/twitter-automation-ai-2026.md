## 实战：如何设计自动化脚本？

### 步骤 1：预设推文库与热点获取
我们不能每天发同样的内容。最稳妥的方式是：**预设素材库 + AI 实时生成**。

你可以建立一个 `tweets.txt`，每天从这里随机抽取一条，加上 AI 改写。
同时，利用脚本每天抓取推特的 Trending（热点趋势），传给 Claude API，让它根据热点写一条带有你博客链接的引流推文。

### 步骤 2：使用浏览器自动化发推
放弃 API，使用类似 `opencli` 这样的现代浏览器自动化工具。
脚本逻辑如下：
1. 唤起 Chrome 浏览器，访问 twitter.com
2. 注入 Cookie 保持登录状态
3. 定位到输入框，填入 AI 生成的文案
4. 模拟点击“Post”按钮

```bash
# 伪代码示例：调用自动化工具发送推文
TWEET_CONTENT="今天发现一个超棒的AI工具组合！看这篇文章👉 https://你的博客.com"
opencli browser run "在输入框中输入：$TWEET_CONTENT，然后点击发送推文按钮"
```

### 步骤 3：定时守护进程
使用 Linux 的 `crontab` 或者自己写一个 Daemon 脚本，设置每天固定时间点运行。

建议的发布频率（UTC+8）：
- 早上 08:00（通勤时间）
- 中午 12:00（午休时间）
- 晚上 20:00（活跃高峰）

## 避坑指南与安全策略

做自动化运营，最怕的就是被封号。请务必遵守以下规则：

1. **频率控制**：每天发推不要超过 5 条，关注别人不要超过 20 个。
2. **随机延迟**：在每步操作之间加入 5-15 秒的随机延迟，模拟真人思考和打字的时间。
3. **不要纯发广告**：维持“提供价值”和“引流”的比例为 3:1 或 4:1。先给读者有用的干货，最后再附带链接。
4. **稳定 IP**：VPS 的 IP 固定后就不要频繁更换。
