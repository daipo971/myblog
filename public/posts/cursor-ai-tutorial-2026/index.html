先说结论#

Cursor 是我用过最好用的 AI 编程工具。但这不是重点，重点是——你得会用。

我第一次用的时候也觉得很一般，跟 GitHub Copilot 差不多嘛。后来才发现是自己没玩对。

我是怎么入坑的#

去年年底被一个推特上的老哥安利的。当时他说"用了Cursor就回不去了"，我还不信。

结果用了两周，真香。

我现在的工作流基本是：

  • Cursor 写代码
  • Claude 做架构和复杂分析
  • ChatGPT 查文档和调试

三个各干各的，效率拉满。

但这套组合是我踩了无数坑试出来的。下面说几个重点。

这几个坑你别踩#

1. 别把 Cursor 当普通编辑器用#

如果你装完 Cursor 就像用 VS Code 一样改代码、按 Ctrl+S、等 Copilot 给提示…那你体验不会好。

Cursor 的核心是 Cmd+K（选中代码直接改）和 Cmd+I（Composer 多文件编辑）。

我见过太多人装了 Cursor 却只用自动补全，暴殄天物。

2. 免费版其实够了#

每天 2000 次补全 + 500 次对话，我重度写一天代码都用不完。

当然如果你是个 996 全栈，或者同时搞好几个项目，Pro 版 $20/月也值。

3. Composer 才是灵魂#

Cmd+I 打开 Composer，你可以一次性让 Cursor 创建/修改好几个文件。

比如我说： “帮我加一个用户登录功能，包括前端页面、API 路由、数据库迁移”

它能一次性给你全搞定。这是 Copilot 做不到的。

我现在的配置#

模型选了 Claude (比 GPT-4 更适合编程)，Rules 里加了项目说明。

最重要的一个技巧：在项目根目录建一个 .cursorrules 文件，写上你项目的技术栈和规范。Cursor 会自动读取，上下文理解会准很多。

我的 .cursorrules 大概长这样：

你是一个资深全栈开发者。
技术栈：Next.js + TypeScript + Prisma + Tailwind
命名规范：文件名小写中划线，函数名驼峰

试一下，差距很大。

哪个AI更适合编程？#

这半年我三个主力AI都在用，个人感受：

  • Cursor：日常写代码最快，上下文理解最好
  • Claude：做架构设计和分析最靠谱
  • ChatGPT：查资料、调 bug、写文档

所以我的建议是：别只用一个。

总结#

Cursor 确实好东西，但不是安装完就完事了。花半小时配置一下 .cursorrules，学会用 Composer，体验会有质的提升。
